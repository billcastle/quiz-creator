# QZ-0001 — Phase 00: Monorepo Scaffold, Tech Stack Installation & Tooling

## Description

Bootstrap the Questify monorepo from scratch. This ticket covers only the initial project setup: create the workspace structure, install all packages, configure all tooling, and verify the base scaffold runs. No feature implementation.

A working dev environment — `npm run dev` starts both the web app and the API with no errors — is the definition of done.

```

  apps/

    api/          ← @quiz/api   — Hono + Cloudflare Workers

    shared/       ← @quiz/shared — Zod schemas + shared TypeScript types

```

## Notes

### Resolved Tech Stack Decisions

**React version:** React 19 (latest stable). The `v16.x.x` spec in the original requirements was EOL — all TanStack packages require 17+.

**Database access:** `better-sqlite3` removed. Drizzle ORM's D1 adapter (`drizzle-orm/d1`) handles all DB reads/writes via the `DB` binding injected by the Workers runtime. This is the only supported approach on Cloudflare Workers (no native Node.js modules, no TCP). For local dev, `wrangler dev` provides a local D1 SQLite proxy automatically — no extra setup.

```typescript
// Correct pattern in Hono handlers
import { drizzle } from 'drizzle-orm/d1'
const db = drizzle(c.env.DB)
```

**Session storage:** Cloudflare KV replaces Redis. Standard Redis clients use TCP, which is unavailable on Workers. better-auth has a first-class KV adapter for sessions. KV is simpler, cheaper, and requires no extra package.

**Email verification:** Skipped for Phase 00 and Phase 01 initial auth. No `resend` or email service in scope for this ticket.

### Resolved Product Decisions

**Questionnaire access model:**
- Any visitor (unauthenticated) can view and answer questionnaires
- Registration is required to create a questionnaire
- No guest-creation flow — the creator must have an account

**Analytics access control:**
- Only the user who created a questionnaire can view its analytics
- Admin analytics page is a separate role (not in scope this phase)
- User roles required in the auth/schema layer: `user`, `admin`

**Shareable links:** nanoid 8-char slug (e.g. `/q/V1StGXR8`). Collision check on insert (retry if slug exists).

**Theme:**
- Default to system preference (`prefers-color-scheme`)
- Persist selected theme in the authenticated user's profile
- Unauthenticated users: localStorage only
