# QZ-0011 — Scaffold packages/ui with Tailwind v4 and shadcn config

## Description

Create the `packages/ui` workspace that will house all design system primitives and domain components. In this phase only the skeleton is created — the actual shadcn components, Tailwind CSS variables, and design tokens are added in PHASE-04.

The package must declare all Radix UI primitives as dependencies (they will be needed when components are added in PHASE-04) along with Tailwind v4, CVA, clsx, and tailwind-merge. The `components.json` configures shadcn/ui to point at this package.

Reference: `docs/architecture/PHASE-03-monorepo-scaffolding.md` — packages/ui Configuration section.

## Acceptance Criteria

- [ ] `packages/ui/package.json` exists with name `@quiz/ui`, all Radix UI dependencies, tailwindcss, clsx, tailwind-merge, class-variance-authority, lucide-react, sonner, and correct exports map
- [ ] `packages/ui/tsconfig.json` extends `../../tsconfig.json` with `lib: ["ES2022", "DOM", "DOM.Iterable"]` and `jsx: react-jsx`
- [ ] `packages/ui/src/index.ts` exists (empty export stub `export {}`)
- [ ] `packages/ui/components.json` exists with shadcn config pointing at this package (`aliases.utils: "@quiz/ui/lib/utils"`, `tailwind.css: "packages/ui/src/globals.css"`)
- [ ] `packages/ui/src/globals.css` exists (empty — populated in PHASE-04)
- [ ] `@quiz/ui` is resolvable from `apps/web` via the path alias
- [ ] `npm run typecheck -w @quiz/ui` passes with zero type errors

## Out of Scope

- Installing shadcn components — PHASE-04
- Tailwind CSS variables and design tokens — PHASE-04
- Any component implementation
