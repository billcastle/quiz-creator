# QZ-0008 — Initialize root package.json and Biome config

## Description

Create the root `package.json` that defines the npm workspaces, shared scripts, Node version constraint, and shared devDependencies. Also create `biome.json` with the agreed formatter, linter, and import organization rules.

The root `package.json` has no runtime dependencies — only `@biomejs/biome`, `@playwright/test`, and `typescript` as devDependencies. All workspace-specific dependencies are declared in each workspace's own `package.json`.

The `biome.json` sets `noExplicitAny: error` to enforce strict typing and `noUnusedVariables/Imports: error` to keep code clean. It replaces ESLint + Prettier entirely.

Reference: `docs/architecture/PHASE-03-monorepo-scaffolding.md` — Root Package Configuration and Biome Configuration sections.

## Acceptance Criteria

- [ ] `package.json` exists at repo root with `workspaces: ["apps/*", "packages/*"]`
- [ ] All 8 scripts defined: `dev`, `dev:web`, `dev:api`, `build`, `lint`, `format`, `typecheck`, `test`
- [ ] `engines: { "node": ">=20.0.0" }` present
- [ ] `biome.json` exists at repo root with formatter (2-space indent, lineWidth 100), linter (recommended + `noExplicitAny: error`, `noUnusedVariables: error`, `noUnusedImports: error`), and `organizeImports: enabled`
- [ ] `files.ignore` in biome.json covers: `node_modules`, `dist`, `.wrangler`, `migrations`, `*.config.js`, `.next`, `coverage`
- [ ] `npx biome check .` runs without config errors
- [ ] `npm install` generates `package-lock.json` at root
