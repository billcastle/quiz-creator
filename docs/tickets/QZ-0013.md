# QZ-0013 — Add CI workflow, root tsconfig, .env.template, and .gitignore

## Description

The final scaffolding ticket. Creates the root `tsconfig.json` (base strict config with `@quiz/*` path aliases), the GitHub Actions CI workflow (lint + typecheck + build jobs), the `.env.template` (all required env var names with empty values), an updated `.gitignore`, and a stub `playwright.config.ts`.

The root `tsconfig.json` does NOT compile any files directly — it is a base config that all workspace `tsconfig.json` files extend. It uses `project references` to point at each workspace.

The CI workflow has three jobs: `lint` (Biome), `typecheck` (tsc --noEmit), and `build` (all workspaces). The `build` job depends on `lint` and `typecheck` passing first.

Reference: `docs/architecture/PHASE-03-monorepo-scaffolding.md` — TypeScript Configuration, GitHub Actions CI, .gitignore, Environment Variables sections.

## Acceptance Criteria

- [ ] `tsconfig.json` at repo root with all strict flags (`strict`, `noUncheckedIndexedAccess`, `exactOptionalPropertyTypes`, `noImplicitOverride`, `noFallthroughCasesInSwitch`), `moduleResolution: "bundler"`, and `@quiz/*` path aliases for all 3 packages
- [ ] `.github/workflows/ci.yml` exists with lint, typecheck, and build jobs; build job `needs: [lint, typecheck]`; web dist uploaded as artifact on main-branch pushes
- [ ] `.env.template` contains all required env var names: `BETTER_AUTH_SECRET`, `BETTER_AUTH_URL`, `CORS_ORIGIN`, `CLOUDFLARE_ACCOUNT_ID`, `CLOUDFLARE_API_TOKEN`, `D1_DATABASE_ID`, `KV_NAMESPACE_ID`, `NODE_ENV`, `VITE_API_URL`
- [ ] `.gitignore` covers `node_modules/`, `dist/`, `.wrangler/`, `*.tsbuildinfo`, `.env`, `.env.local`, `.DS_Store`, `test-results/`, `playwright-report/`, `coverage/`
- [ ] `playwright.config.ts` stub exists at repo root
- [ ] `npm run typecheck` passes with zero errors across all workspaces
- [ ] `npm run build` completes successfully for all packages and apps
- [ ] Full CI locally: `npm run lint && npm run typecheck && npm run build` all pass
- [ ] Pushed to remote — GitHub Actions CI runs and passes all three jobs
