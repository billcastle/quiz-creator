# QZ-0009 — Scaffold apps/web with Vite + React 19

## Description

Create the `apps/web` workspace with all configuration files and a stub `main.tsx`. This workspace uses Vite + React 19 + TypeScript as the bundler and framework (ADR-001), TanStack Router for routing (ADR-002), TanStack Query for server state (ADR-003), shadcn/ui via `@quiz/ui` for components (ADR-004), and Zustand for client state (ADR-005).

The Vite config must include the TanStack Router plugin (which auto-generates the route tree), the React plugin, and path aliases for `@` (local src) and `@quiz/ui`, `@quiz/shared` (pointing at the packages).

The dev server proxies `/api` to `http://localhost:8787` so the frontend and backend can communicate during local development.

Reference: `docs/architecture/PHASE-03-monorepo-scaffolding.md` — apps/web Configuration section.

## Acceptance Criteria

- [ ] `apps/web/package.json` exists with name `@quiz/web`, all dependencies listed in the blueprint, and scripts: `dev`, `build`, `preview`, `typecheck`
- [ ] `apps/web/vite.config.ts` exists with TanStack Router plugin, React plugin, path aliases for `@`, `@quiz/ui`, `@quiz/shared`, build target `esnext`, and dev server proxy `/api → http://localhost:8787`
- [ ] `apps/web/tsconfig.json` extends `../../tsconfig.json` with `lib: ["ES2022", "DOM", "DOM.Iterable"]` and `jsx: react-jsx`
- [ ] `apps/web/index.html` exists as standard Vite entry point referencing `src/main.tsx`
- [ ] `apps/web/src/main.tsx` renders a stub React app (no routing setup yet — just confirms React renders)
- [ ] `npm run dev:web` starts Vite dev server on port 5173 without errors after `npm install`
- [ ] `npm run typecheck -w @quiz/web` passes with zero type errors
