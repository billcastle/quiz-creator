# QZ-0006 — Phase 02: Questionnaire Builder

## Description

Implement the questionnaire creation and editing experience. This ticket extends the domain schema from QZ-0005 with the shared supporting tables (`sections`, `questions`, `question_options`), adds a full CRUD API for questionnaires and their questions, and replaces the `QuizBuilderPage` stub with a functional builder UI.

A creator should be able to open the builder from `/quiz/new` or `/quiz/:id/edit`, set the questionnaire title and settings, add sections (optional), add questions of any supported type, reorder questions, and save. The questionnaire is saved as a draft until explicitly published.

The taker-facing experience is out of scope for this ticket (see QZ-0008).

## DB Schema

Add to `packages/db/src/schema.ts` (per ADR-009):

```ts
export const sections = sqliteTable('sections', {
  id: text('id').primaryKey(),
  parentType: text('parent_type', { enum: ['questionnaire', 'survey'] }).notNull(),
  parentId: text('parent_id').notNull(),
  title: text('title').notNull().default(''),
  position: integer('position').notNull().default(0),
  createdAt: integer('created_at', { mode: 'timestamp' }).notNull(),
  updatedAt: integer('updated_at', { mode: 'timestamp' }).notNull(),
})

export const questions = sqliteTable('questions', {
  id: text('id').primaryKey(),
  parentType: text('parent_type', { enum: ['questionnaire', 'survey'] }).notNull(),
  parentId: text('parent_id').notNull(),
  sectionId: text('section_id'),  // nullable — question may not belong to a section
  type: text('type', { enum: ['single_choice', 'multiple_choice', 'short_answer', 'long_answer'] }).notNull(),
  prompt: text('prompt').notNull().default(''),
  position: integer('position').notNull().default(0),
  required: integer('required', { mode: 'boolean' }).notNull().default(true),
  // Questionnaire-only fields (ignored for surveys)
  showCorrectAnswer: integer('show_correct_answer', { mode: 'boolean' }).notNull().default(false),
  caseSensitive: integer('case_sensitive', { mode: 'boolean' }).notNull().default(false),
  acceptableAnswers: text('acceptable_answers'),  // JSON string: string[]
  createdAt: integer('created_at', { mode: 'timestamp' }).notNull(),
  updatedAt: integer('updated_at', { mode: 'timestamp' }).notNull(),
})

export const questionOptions = sqliteTable('question_options', {
  id: text('id').primaryKey(),
  questionId: text('question_id').notNull().references(() => questions.id, { onDelete: 'cascade' }),
  label: text('label').notNull().default(''),
  position: integer('position').notNull().default(0),
  isCorrect: integer('is_correct', { mode: 'boolean' }).notNull().default(false),
})
```

After adding the tables, run:
```bash
npm run db:generate
npm run db:migrate:local
```

## API Endpoints

All endpoints require an authenticated session (same pattern as QZ-0005 — call `auth.api.getSession`, return 401 if missing).

### Questionnaire CRUD

| Method | Path | Description |
|---|---|---|
| `POST` | `/api/questionnaires` | Create a new questionnaire. Body: `{ title?: string }`. Returns the created row. |
| `GET` | `/api/questionnaires/:id` | Get a single questionnaire with its sections and questions (nested). Returns 404 if not found or not owned by caller. |
| `PUT` | `/api/questionnaires/:id` | Update questionnaire fields: `title`, `status`, `visibility`, `timeLimitSeconds`, `allowMultipleAttempts`. Returns updated row. |
| `DELETE` | `/api/questionnaires/:id` | Delete questionnaire (cascades to sections, questions, options). Returns 204. |

### Question CRUD

| Method | Path | Description |
|---|---|---|
| `POST` | `/api/questionnaires/:id/questions` | Add a question. Body: `{ type, prompt?, sectionId?, position? }`. Returns created question + empty options array. |
| `PUT` | `/api/questions/:id` | Update question fields: `prompt`, `required`, `showCorrectAnswer`, `caseSensitive`, `acceptableAnswers`. Returns updated question. |
| `DELETE` | `/api/questions/:id` | Delete question. Returns 204. |
| `PUT` | `/api/questionnaires/:id/questions/reorder` | Reorder all questions. Body: `{ order: string[] }` (array of question IDs in new position order). |

### Question Options

| Method | Path | Description |
|---|---|---|
| `POST` | `/api/questions/:id/options` | Add an option. Body: `{ label?, isCorrect? }`. Returns created option. |
| `PUT` | `/api/question-options/:id` | Update option: `label`, `isCorrect`. Returns updated option. |
| `DELETE` | `/api/question-options/:id` | Delete option. Returns 204. |

## Builder UI (`QuizBuilderPage`)

Replace the stub at `apps/web/src/pages/QuizBuilderPage.tsx`.

### Layout

Two-column layout at `lg` breakpoint:
- **Left panel** (scrollable): question list with add button at bottom. Each question is a card showing its type badge and truncated prompt. Active question is highlighted. Drag handle for reorder (CSS-only drag is fine at this stage; no drag library required).
- **Right panel** (fixed): question editor for the currently selected question. Shows prompt textarea, type selector, and type-specific fields (options list for choice types, acceptable answers for short_answer).
- On mobile: single-column, question list above, editor below.

### Header

- Editable title (inline `<input>` or `contenteditable`, auto-saves on blur).
- Status badge (Draft / Published / Archived).
- "Save" button (manual save alongside auto-save on blur).
- "Publish" button — changes `status` to `published` and saves.
- Back chevron → navigates to `/` with a confirmation dialog if unsaved changes exist.

### Question Card (left panel)

- Type badge (`single_choice`, `multiple_choice`, `short_answer`, `long_answer`).
- Truncated prompt text (or "Untitled question" placeholder).
- Delete icon button.
- Drag handle (visual only at this stage).

### Question Editor (right panel)

- **Type selector**: four-option segmented control or radio group.
- **Prompt**: `<textarea>` (plain text; TipTap rich text is deferred to a later ticket per ADR-007).
- **Required toggle**: checkbox.
- **Type-specific fields**:
  - `single_choice` / `multiple_choice`: editable option list. Each option has a label input, a "correct" toggle (checkbox for multiple_choice, radio for single_choice), and a delete button. "+ Add option" button appends a new blank option.
  - `short_answer`: "Acceptable answers" — tag input or comma-separated text input. "Case sensitive" toggle.
  - `long_answer`: no additional fields.

### Data flow

- On mount: if route has `$id` param, `GET /api/questionnaires/:id` to load. Otherwise call `POST /api/questionnaires` immediately to create a new draft and push `{ to: '/quiz/$id/edit', params: { id } }` to update the URL.
- Auto-save on blur of title input (debounce 500 ms or immediate on-blur).
- Save button: `PUT /api/questionnaires/:id` with current state.
- Question edits auto-save on blur of each field (500 ms debounce).
- Optimistic UI: update local state immediately, fire the API call in the background. On error, revert local state and show a toast.

## Notes

- Use the same `crypto.randomUUID()` pattern for IDs as Better Auth uses (`text('id').primaryKey()`).
- `sections` and `questions` use a `parent_type` + `parent_id` polymorphic FK (per ADR-009 NEG-001). Enforce at the application layer before insert.
- `acceptableAnswers` is stored as a JSON string (`JSON.stringify(string[])`) because D1/SQLite has no native array type.
- The `position` integer on questions starts at 0 and is contiguous. The reorder endpoint accepts the new ordered array of IDs and updates all positions in a transaction.
