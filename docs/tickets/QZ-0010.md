# QZ-0010 — Scaffold apps/api with Hono + Wrangler

## Description

Create the `apps/api` workspace targeting Cloudflare Workers. This workspace uses Hono as the backend framework (ADR-006) and Wrangler as the deployment tool. The only route in this phase is `/health` — all business logic routes are added in later phases.

The `wrangler.toml` must declare the D1 database binding (`QUIZ_DB`) and KV namespace binding (`QUIZ_KV`) with empty IDs (to be filled in during PHASE-16 when Cloudflare resources are provisioned). The `nodejs_compat` flag is required for Better-auth compatibility.

The Hono stub defines the `Bindings` type inline — this will be extracted to a shared file in PHASE-08.

Reference: `docs/architecture/PHASE-03-monorepo-scaffolding.md` — apps/api Configuration section.

## Acceptance Criteria

- [ ] `apps/api/package.json` exists with name `@quiz/api`, dependencies (hono, better-auth, @quiz/db, @quiz/shared) and devDependencies (@cloudflare/workers-types, wrangler, typescript)
- [ ] `apps/api/wrangler.toml` exists with name `quiz-api`, `nodejs_compat: true`, D1 binding `QUIZ_DB`, KV binding `QUIZ_KV`, and staging/production env stubs
- [ ] `apps/api/tsconfig.json` extends `../../tsconfig.json` with `types: ["@cloudflare/workers-types"]`
- [ ] `apps/api/src/index.ts` exports a Hono app with the `Bindings` type and a single `GET /health` route returning `{ status: "ok", phase: "scaffolding" }`
- [ ] `npm run dev:api` starts Wrangler dev on port 8787 without errors
- [ ] `GET http://localhost:8787/health` returns `{"status":"ok","phase":"scaffolding"}`
- [ ] `npm run typecheck -w @quiz/api` passes with zero type errors

## Out of Scope

- Authentication routes — PHASE-09
- Business logic routes — PHASE-08 onwards
- Cloudflare resource provisioning (filling in D1/KV IDs) — PHASE-16
