# QZ-0012 — Scaffold packages/db and packages/shared

## Description

Create two shared packages: `@quiz/db` (Drizzle ORM + D1 client) and `@quiz/shared` (Zod schemas, types, constants).

`packages/db` uses Drizzle ORM with the `d1-http` driver (ADR-008). The `drizzle.config.ts` points at the D1 HTTP driver using Cloudflare API credentials from env vars. Schema files are stubs — actual tables are defined in PHASE-06.

`packages/shared` has Zod as its only runtime dependency. It is the dependency floor: it must never import from `apps/*` or other packages. `packages/db` may import from it. All other packages and apps may import from it.

Reference: `docs/architecture/PHASE-03-monorepo-scaffolding.md` — packages/db Configuration and packages/shared Configuration sections.

## Acceptance Criteria

- [ ] `packages/db/package.json` exists with name `@quiz/db`, `drizzle-orm` dependency, `drizzle-kit` devDependency, and DB scripts: `db:generate`, `db:migrate:local`, `db:migrate:prod`, `db:studio`
- [ ] `packages/db/drizzle.config.ts` exists using `dialect: "sqlite"`, `driver: "d1-http"`, schema pointing at `./src/schema/index.ts`
- [ ] `packages/db/tsconfig.json` extends root with `types: ["@cloudflare/workers-types"]`
- [ ] `packages/db/src/index.ts` and `packages/db/src/schema/index.ts` exist as stubs (`export {}`)
- [ ] `packages/shared/package.json` exists with name `@quiz/shared`, only `zod` as runtime dependency, and exports map for `.`, `./types`, `./schemas`, `./constants`
- [ ] `packages/shared/tsconfig.json` extends root
- [ ] `packages/shared/src/index.ts`, `src/types/index.ts`, `src/schemas/index.ts`, `src/constants/index.ts` all exist as stubs
- [ ] Both packages typecheck cleanly after `npm install` from root

## Out of Scope

- Drizzle schema definitions (tables) — PHASE-06
- Zod schemas for domain types — PHASE-06
- Drizzle migrations — PHASE-06
