# QZ-0005 — Phase 02: Creator Homepage

## Description

Replace the `HomePage.tsx` placeholder with the creator's content dashboard. The page lists all questionnaires and surveys the authenticated user has created, lets them filter by type, and provides entry points into the builder. An empty state guides new users toward creating their first piece of content.

This ticket also lays the database foundation for the domain model defined in ADR-009: the `questionnaires` and `surveys` tables, and the API list endpoints those tables require. The builder pages (`/questionnaire/new`, `/survey/new`) remain stubs — full builder implementation is QZ-0006 and QZ-0007.

Definition of done: a signed-in creator sees their content list (or empty state), can filter between Questionnaires and Surveys, and can navigate to the builder stubs. `npm run typecheck` and `npm run lint` both pass.

## Database Schema

Add two tables to `packages/db/src/schema.ts` per ADR-009.

### `questionnaires`

```ts
export const questionnaires = sqliteTable('questionnaires', {
  id: text('id').primaryKey(),
  creatorId: text('creator_id').notNull().references(() => user.id, { onDelete: 'cascade' }),
  title: text('title').notNull().default('Untitled Questionnaire'),
  status: text('status', { enum: ['draft', 'published', 'archived'] }).notNull().default('draft'),
  visibility: text('visibility', { enum: ['public', 'private'] }).notNull().default('private'),
  timeLimitSeconds: integer('time_limit_seconds'),
  allowMultipleAttempts: integer('allow_multiple_attempts', { mode: 'boolean' }).notNull().default(false),
  createdAt: integer('created_at', { mode: 'timestamp' }).notNull(),
  updatedAt: integer('updated_at', { mode: 'timestamp' }).notNull(),
})
```

### `surveys`

```ts
export const surveys = sqliteTable('surveys', {
  id: text('id').primaryKey(),
  creatorId: text('creator_id').notNull().references(() => user.id, { onDelete: 'cascade' }),
  title: text('title').notNull().default('Untitled Survey'),
  status: text('status', { enum: ['draft', 'published', 'archived'] }).notNull().default('draft'),
  visibility: text('visibility', { enum: ['public', 'private'] }).notNull().default('private'),
  allowMultipleAttempts: integer('allow_multiple_attempts', { mode: 'boolean' }).notNull().default(false),
  createdAt: integer('created_at', { mode: 'timestamp' }).notNull(),
  updatedAt: integer('updated_at', { mode: 'timestamp' }).notNull(),
})
```

After updating `schema.ts`, generate and apply the migration:

```bash
npx drizzle-kit generate
npx wrangler d1 migrations apply QUIZ_DB --local
```

## API Layer (`apps/api`)

Both endpoints require an authenticated session. Use Better Auth's `auth.api.getSession` to extract the caller's user ID. Return 401 if no session.

### `GET /api/questionnaires`

Returns the authenticated creator's questionnaires, newest first.

**Response:**
```json
{
  "questionnaires": [
    {
      "id": "abc123",
      "title": "Week 3 Quiz",
      "status": "draft",
      "visibility": "private",
      "createdAt": 1751260800
    }
  ]
}
```

**Route handler sketch:**

```ts
app.get('/api/questionnaires', async (c) => {
  const auth = createAuth(c.env)
  const session = await auth.api.getSession({ headers: c.req.raw.headers })
  if (!session) return c.json({ error: 'Unauthorized' }, 401)

  const db = createDb(c.env.QUIZ_DB)
  const rows = await db
    .select()
    .from(questionnaires)
    .where(eq(questionnaires.creatorId, session.user.id))
    .orderBy(desc(questionnaires.createdAt))

  return c.json({ questionnaires: rows })
})
```

### `GET /api/surveys`

Same shape as the questionnaires endpoint, against the `surveys` table.

## Web Layer (`apps/web`)

### API client utility

**File:** `apps/web/src/lib/api.ts`

A thin typed wrapper around `fetch` that uses the session cookie automatically (same-origin, so no extra headers needed). Keep it minimal — this is not an SDK.

```ts
async function get<T>(path: string): Promise<T> {
  const res = await fetch(path, { credentials: 'include' })
  if (!res.ok) throw new Error(`${res.status} ${res.statusText}`)
  return res.json() as Promise<T>
}

export const api = { get }
```

### `HomePage.tsx`

Replace the placeholder with the creator dashboard.

#### Layout

```
<HomePage>
  <header>
    <h1>My Content</h1>
    <Button onClick={() => navigate('/questionnaire/new')}>+ New Questionnaire</Button>
    <Button onClick={() => navigate('/survey/new')}>+ New Survey</Button>
  </header>

  <Tabs value={filter} onValueChange={setFilter}>
    <TabsList>
      <TabsTrigger value="all">All</TabsTrigger>
      <TabsTrigger value="questionnaires">Questionnaires</TabsTrigger>
      <TabsTrigger value="surveys">Surveys</TabsTrigger>
    </TabsList>
  </Tabs>

  {isLoading && <LoadingSkeleton />}
  {isEmpty && <EmptyState type={filter} />}
  {!isEmpty && <ContentGrid items={filteredItems} />}
</HomePage>
```

#### Data fetching

Use `useEffect` + `useState` to fetch both lists on mount. No external query library — keep it simple until a data-fetching pattern is established.

```ts
const [questionnaires, setQuestionnaires] = useState<Questionnaire[]>([])
const [surveys, setSurveys] = useState<Survey[]>([])
const [isLoading, setIsLoading] = useState(true)

useEffect(() => {
  Promise.all([
    api.get<{ questionnaires: Questionnaire[] }>('/api/questionnaires'),
    api.get<{ surveys: Survey[] }>('/api/surveys'),
  ])
    .then(([q, s]) => {
      setQuestionnaires(q.questionnaires)
      setSurveys(s.surveys)
    })
    .finally(() => setIsLoading(false))
}, [])
```

#### Content card

Each item in the grid is a card showing:
- Title (truncated to one line)
- Type badge: `Questionnaire` (violet, `--color-quiz`) or `Survey` (sky, `--color-survey`) — use the `Badge` component from QZ-0002
- Status badge: `draft` / `published` / `archived`
- Created-at date formatted as `MMM D, YYYY` (use `Intl.DateTimeFormat`, no date library)
- "Edit" link → `/questionnaire/:id/edit` or `/survey/:id/edit`

Cards link to the edit route. The edit and take routes are still stubs — clicking through to them is fine.

#### Empty state

When the filtered list is empty:
- Illustration: a simple SVG placeholder (box with a plus sign) or a Lucide icon (`FilePlus2` or `ClipboardList`)
- Headline: "No questionnaires yet" / "No surveys yet" / "No content yet"
- Sub-copy: "Create your first one to get started."
- CTA button: "New Questionnaire" or "New Survey" (matching the filter; "New Questionnaire" as default when `all` is selected)

#### Loading state

Show 3 skeleton card placeholders while data is fetching. Use `animate-pulse` with `bg-[var(--color-bg-subtle)]` rounded boxes — no external skeleton library.

## Notes

### Auth guard on API endpoints

Better Auth's `auth.api.getSession` reads the `better-auth.session_token` cookie from the request headers. Because the Vite proxy forwards the cookie on the same origin, `credentials: 'include'` on the `fetch` call is sufficient — no explicit `Authorization` header needed.

### No query library yet

`useEffect` + `useState` is sufficient for a two-request homepage fetch. A data-fetching library (React Query, SWR) is not introduced here — add it when the pattern is proven or when more pages need it.

### Tabs component

Use shadcn `Tabs` / `TabsList` / `TabsTrigger` (already available from QZ-0002's shadcn foundation). Filter state is local to `HomePage` — no URL param needed at this stage.
