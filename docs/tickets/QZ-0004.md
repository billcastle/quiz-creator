# QZ-0004 — Phase 02: Authentication (Better Auth, Sign-in & Sign-up)

## Description

Replace the `auth-stub.ts` localStorage hack from QZ-0003 with real email+password authentication using Better Auth. This ticket wires the full auth stack end-to-end: D1-backed user records, KV-backed session tokens, Hono route handlers, a typed web client, and real sign-in/sign-up form pages.

The auth stub surface is small by design (one file, one `beforeLoad` hook, two placeholder pages). Replacing it is a contained swap — not a refactor of the shell or navigation built in QZ-0003.

Definition of done: a new user can register with an email and password, sign in and receive a session cookie, navigate the authenticated shell, and sign out. `npm run typecheck` and `npm run lint` both pass. The `auth-stub.ts` file is deleted.

## Database Schema Changes

Better Auth requires four tables. Add them to `packages/db/src/schema.ts` using the Drizzle SQLite DSL. Better Auth's CLI can generate these: `npx @better-auth/cli generate --adapter drizzle`. Review and commit the generated output — do not add them by hand.

| Table | Purpose |
|---|---|
| `user` | Registered users — `id`, `email`, `emailVerified`, `createdAt`, `updatedAt` |
| `session` | Active sessions — `id`, `userId`, `token`, `expiresAt`, `createdAt`, `updatedAt` |
| `account` | OAuth / credential accounts linked to a user — `id`, `userId`, `accountId`, `providerId`, `accessToken`, `refreshToken`, `expiresAt` |
| `verification` | Email verification tokens — `id`, `identifier`, `value`, `expiresAt` |

After schema is updated, generate and apply the migration:

```bash
# Generate SQL from schema diff
npx drizzle-kit generate

# Apply to local D1 dev database
npx wrangler d1 migrations apply QUIZ_DB --local
```

## API Layer (`apps/api`)

### Install

```bash
npm install better-auth --workspace=apps/api
```

### Auth instance factory

Per ADR-002 IMP-002: Better Auth must **not** be initialized at module level. It requires Cloudflare env bindings (`QUIZ_DB`, `QUIZ_KV`) that are only available at request time. Create a factory:

**File:** `apps/api/src/lib/auth.ts`

```ts
import { betterAuth } from 'better-auth'
import { drizzleAdapter } from 'better-auth/adapters/drizzle'
import { createDb } from '@questify/db'

// Called once per request — receives c.env from the Hono handler
export function createAuth(env: { QUIZ_DB: D1Database; QUIZ_KV: KVNamespace; BETTER_AUTH_SECRET: string }) {
  return betterAuth({
    secret: env.BETTER_AUTH_SECRET,
    database: drizzleAdapter(createDb(env.QUIZ_DB), { provider: 'sqlite' }),
    secondaryStorage: {
      // KV-backed session storage per ADR-004
      get: (key) => env.QUIZ_KV.get(key),
      set: (key, value, ttl) => env.QUIZ_KV.put(key, value, ttl ? { expirationTtl: ttl } : undefined),
      delete: (key) => env.QUIZ_KV.delete(key),
    },
    emailAndPassword: { enabled: true },
    trustedOrigins: [env.CORS_ORIGIN],
  })
}
```

> **Note:** If Better Auth's KV adapter signature differs from the above, consult the [Better Auth Cloudflare Workers guide](https://www.better-auth.com/docs/integrations/cloudflare) and adapt accordingly.

### Hono route handler

Mount Better Auth's handler at `POST /api/auth/**` in `apps/api/src/index.ts`. Better Auth exposes a single handler that covers all auth sub-routes (sign-in, sign-up, sign-out, session, etc.):

```ts
app.all('/api/auth/**', (c) => {
  const auth = createAuth(c.env)
  return auth.handler(c.req.raw)
})
```

Verify with `wrangler dev`:
- `POST /api/auth/sign-up/email` → 200 creates user
- `POST /api/auth/sign-in/email` → 200 sets session cookie
- `GET /api/auth/get-session` → 200 returns session or `null`
- `POST /api/auth/sign-out` → 200 clears session cookie

### Bindings (`apps/api/src/index.ts`)

`CORS_ORIGIN` and `BETTER_AUTH_SECRET` are already in the `Bindings` type. No change needed.

### `wrangler.toml`

Confirm `[[kv_namespaces]]` with `binding = "QUIZ_KV"` and a valid `id` for both dev and production. If placeholder IDs exist, create real namespaces:

```bash
npx wrangler kv namespace create QUIZ_KV
npx wrangler kv namespace create QUIZ_KV --preview
```

## Web Layer (`apps/web`)

### Install

```bash
npm install better-auth --workspace=apps/web
```

Better Auth ships its own client package — no separate `@better-auth/client` install needed.

### Auth client

**File:** `apps/web/src/lib/auth-client.ts`

```ts
import { createAuthClient } from 'better-auth/client'

export const authClient = createAuthClient({
  baseURL: import.meta.env.VITE_API_URL ?? 'http://localhost:8787',
})
```

Add `VITE_API_URL` to `apps/web/.env.local` pointing at `wrangler dev` port:

```
VITE_API_URL=http://localhost:8787
```

### Replace auth-stub

1. **Delete** `apps/web/src/lib/auth-stub.ts`
2. Update `apps/web/src/router.tsx` — replace the `beforeLoad` guard:

```ts
// Before (stub):
beforeLoad: () => {
  if (!isAuthenticatedStub()) throw redirect({ to: '/sign-in' })
}

// After (real):
beforeLoad: async () => {
  const { data: session } = await authClient.getSession()
  if (!session) throw redirect({ to: '/sign-in' })
}
```

3. Update `apps/web/src/layouts/AuthenticatedLayout.tsx` — replace `stubSignOut()` in `handleSignOut`:

```ts
async function handleSignOut() {
  await authClient.signOut()
  navigate({ to: '/sign-in' })
}
```

### Sign-in page (`apps/web/src/pages/SignInPage.tsx`)

Replace the stub button with a real email+password form:

| Field | Type | Validation |
|---|---|---|
| Email | `<input type="email">` | required, valid email format |
| Password | `<input type="password">` | required, min 8 characters |
| Submit | `<Button>` | "Sign in" |
| Error state | inline text | show Better Auth error message below form |
| Link | `<button>` → `/sign-up` | "Don't have an account? Sign up" |

On submit, call `authClient.signIn.email({ email, password })`. On success, navigate to `/`. On error, display the returned error message.

Use the design token classes (`var(--color-*)`) established in QZ-0002 for all form styling.

## Notes

### Per-request auth instance

Better Auth cannot be initialized at module load time in Workers because `QUIZ_DB`, `QUIZ_KV`, and `BETTER_AUTH_SECRET` are not available until the first request. See ADR-002 IMP-002 and ADR-004 IMP-002. The `createAuth(env)` factory pattern ensures the auth instance is constructed with the correct per-request env object.

### Cookie configuration

Better Auth sets a `better-auth.session_token` cookie by default. In development (`wrangler dev` on port 8787, Vite on port 5173), the cookie `SameSite` policy may block cross-origin requests. Configure `sameSite: "none"` and `secure: false` in Better Auth's cookie options for the dev environment, or proxy the API through Vite to avoid the cross-origin cookie issue.

### Better Auth version pinning

Pin to `better-auth@^1.0.0` (or the current stable release). Check the [Better Auth changelog](https://better-auth.com) for breaking changes before upgrading.
