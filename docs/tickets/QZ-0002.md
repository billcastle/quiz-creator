# QZ-0002 — Phase 01: Design System, Themes & /design-system Page

## Description

Bootstrap the Questify design system inside `packages/ui`. This covers:

1. **shadcn/ui installs** — install the shadcn components already in the library; do not rebuild what exists.
2. **Custom components** — build only what shadcn does not provide, derived from `docs/ui/questionnaire-wireframes.pdf`.
3. **Theme engine** — each theme is a standalone `.ts` file in `packages/ui/src/themes/`. Adding a new theme requires only creating one new file — no other changes. Themes are auto-discovered via `import.meta.glob` and applied to the DOM with `document.documentElement.style.setProperty()`. Three themes ship in this ticket: Default, Dark, Dusk.
4. **ThemeToggle component** — dropdown that auto-lists all discovered themes with color swatch previews. Built on shadcn `DropdownMenu`.
5. **`/design-system` route** — a gallery page in `apps/web` that renders every component in every variant, with a sticky ThemeToggle in the top-right corner.

The definition of done: `npm run dev:web` → `/design-system` → all components visible, all three themes apply instantly, `npm run typecheck` and `npm run lint` both pass.

## Theme System

### File-based design

Each theme is a single `.ts` file in `packages/ui/src/themes/`. Adding a new theme means creating one file — nothing else changes.

```
packages/ui/src/themes/
  types.ts       ← Theme interface (required shape every theme must satisfy)
  index.ts       ← auto-discovery via import.meta.glob; exports themes[]
  default.ts     ← Default (light) theme
  dark.ts        ← Dark theme
  dusk.ts        ← Dusk theme (custom palette)
```

### Theme interface (`types.ts`)

```ts
export interface Theme {
  id: string
  name: string
  /** Five hex colors shown as swatches in ThemeToggle: [bg, surface, accent, text, border] */
  swatches: [string, string, string, string, string]
  tokens: {
    '--color-bg-base': string
    '--color-bg-surface': string
    '--color-bg-subtle': string
    '--color-border': string
    '--color-border-focus': string
    '--color-text-primary': string
    '--color-text-secondary': string
    '--color-text-disabled': string
    '--color-accent': string
    '--color-accent-hover': string
    '--color-accent-fg': string
    '--color-destructive': string
    '--color-success': string
    '--color-warning': string
    '--color-quiz': string
    '--color-survey': string
    '--color-exam': string
  }
}
```

### Auto-discovery (`index.ts`)

```ts
import type { Theme } from './types'

const modules = import.meta.glob<{ default: Theme }>('./*.ts', { eager: true })

export const themes: Theme[] = Object.values(modules)
  .map(m => m.default)
  .filter(Boolean)
  .sort((a, b) => a.name.localeCompare(b.name))

export type { Theme }
```

### Applying a theme

Themes are applied via `style.setProperty` on `<html>` — no CSS class switching required:

```ts
export function applyTheme(theme: Theme): void {
  for (const [key, value] of Object.entries(theme.tokens)) {
    document.documentElement.style.setProperty(key, value)
  }
}
```

CSS in components uses `var(--color-accent)` etc. — works regardless of how the variable was set.

### Theme store (`packages/ui/src/stores/theme.ts`)

```ts
interface ThemeStore {
  activeThemeId: string           // matches Theme.id; not a hardcoded union
  setTheme: (id: string) => void
}
```

Initial load order: `localStorage["questify-theme"]` → `prefers-color-scheme` (dark → `"dark"`, else `"default"`).

--color-text-disabled   disabled state
--color-accent          primary action color

## ThemeToggle Spec

```tsx
<ThemeToggle />
```

- Reads `themes` from auto-discovery (`import.meta.glob`). No hardcoded theme list.
- **Trigger button**: active theme name + 3 color dots (accent, bg-surface, text-primary).
- **Dropdown rows** (one per theme): theme name · 5-swatch color strip · checkmark on active theme.
- **On select**: calls `setTheme(id)` → `applyTheme(theme)` → saves `id` to `localStorage["questify-theme"]`.
- **On mount**: loads saved id; resolves to Theme object from `themes[]`; applies it. Falls back to `"default"`.

## Notes

### Why file-based themes over CSS classes

CSS class themes (`.theme-dark { ... }`) require every theme to be pre-declared in a CSS file. Adding a theme means editing that file. The file-based approach instead stores token values in `.ts` files and applies them with `style.setProperty`. Adding a theme = one new file. The `import.meta.glob` auto-discovery means neither `index.ts` nor `ThemeToggle` need to change.

### shadcn Badge extension pattern

shadcn `Badge` uses `cva` for variants. Add `quiz`, `survey`, `exam` variants that read theme tokens:

```ts
const badgeVariants = cva('...base classes...', {
  variants: {
    variant: {
      // shadcn defaults...
      quiz:   'bg-[var(--color-quiz)] text-[var(--color-accent-fg)]',
      survey: 'bg-[var(--color-survey)] text-[var(--color-accent-fg)]',
      exam:   'bg-[var(--color-exam)] text-[var(--color-accent-fg)]',
    },
  },
})
```

### RadioOption card style

Full-width clickable row wrapping `RadioGroup.Item`. Hidden native radio inside a styled `<label>` so the entire row is the click target. `correct`/`incorrect` states are visual-only — set by the parent component after answer submission, not derived from the radio's checked state.

### Dusk theme name

`["bcf8ec","aed9e0","9fa0c3","8b687f","7b435b"]` transitions from cool mint/sky-blue through periwinkle into warm mauve and deep plum. Theme ID: `"dusk"`, display name: `"Dusk"`.
